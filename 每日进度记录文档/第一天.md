# 第一天

- 决定选题：智能笔记

## 分析需求

- 软件的目的是在保留**快速记录**的方便性的前提下，让 AI 辅助整理笔记，重点是让用户杂乱的笔记重新变得有序，并且能够智能总结，达到让用户能够定期进行知识复盘的效果



需要实现的 AI 功能如下：

- 帮助用户将快速记录的杂乱笔记重新整理，自动分类等
- 希望 AI 能帮助用户回顾笔记，知识复用
- 添加 AI 帮帮功能，用户可以在单独的界面让 AI 辅助问答或者直接操作本地笔记（一定要备份，删除功能最好需要用户确认）
- 整理笔记、形成结构：让 AI 根据用户笔记内容，能够智能分析并梳理大纲，点击即可一键跳转响应知识点，形成知识体系



## 如何实现

### 1. 用户信息

- 用户可以选择**手机号+密码**或者**邮箱+密码**登录，出场自带随机用户名（或者加页面提前让用户取名）
  - 密码在前端后端同时进行校验，暂时限制 [8, 20] 长度，数字、大小写字母、特殊符号三选二
- 用户信息界面可以自定义用户名、头像、座右铭以及修改密码界面

### 2. 笔记管理

- 主界面是笔记管理界面，提供快捷键或者按钮等快速新建笔记
- 显示笔记界面提供根据时间排序功能（如：最新...）
- 主界面拥有搜索框，可以使用标签查询或标题查询等
- 单击笔记进入笔记编辑界面，提供编辑、阅读模式（或者一体化）
- 笔记大纲界面提供 AI 总结服务
- 笔记右键提供重命名、删除等功能
  - 删除使用字段标记已删除
- 用户笔记隔离，绝对不能出现一个用户可以访问另一个用户笔记的情况
- 标签功能：用户使用`#java`等字段即可智能捕捉并添加到笔记的标签中
- 历史记录：查看之前看过的笔记（限制一个月）

### 3. 好友功能

- 可以通过**手机号/邮箱**搜索找到对应用户，并提交好友申请
- 可以给好友添加标签（默认标签和自定义标签）
- 同意后可以添加为好友

### 4. 笔记分享

- 笔记有状态需求：
  - 仅自己可见（默认）
  - 部分好友可见，但是无法编辑
  - 部分好友可见且可编辑
  - 所有人可见但是仅自己可编辑
- 笔记分享功能：
  - 自动生成分享链接
  - 或者复制当前页面 url

### 5. AI 功能

- 大纲页面提供 AI 选项：生成摘要、要点提取、标签建议等
- AI 回复记录到数据库并与笔记关联
- 可以实现重新分析功能（覆盖原记录）

### 6. 注意

- 报错友好
- 鉴权机制完善
- 记录日志
- 架构合理，耦合性低
- 避免硬编码，使用常量类或者枚举类
- 邮箱或者手机号使用正则校验

### 7. 进阶功能

- 笔记文件夹功能
- 版本管理
- 导入 .md 和 .pdf，导出为 .md 或 .pdf
- 前端使用 Markdown 语法







## 建表

使用 MySQL 和 Mybatis Plus

1. 用户信息表

   - id（主键）

   - 用户名（唯一）
   - 手机号（默认，必须有，唯一）
   - 邮箱（可选，可以尝试发送验证码等，唯一）
   - 密码（加密后）
   - 头像（暂存本地，映射到 url 上，加上默认头像）
   - 座右铭（个性签名，可为空）
   - 是否被封禁（0-正常 1-被封禁）
   - 是否被删除（逻辑删除）
   - 注册时间
   - 用户信息最后修改时间

2. 笔记表

   - id
   - 所属账户（不可更改）
   - 文件夹id（默认0）
   - 笔记标题
   - 笔记正文
   - 可见性
   - 标签（存 JSON 或用逗号分隔）
   - 是否被删除（逻辑删除）
   - 最后一次进入回收站的时间（过5分钟物理删除）
   - 创建时间
   - 最后一次更新时间

3. 好友关系表（一个数据只代表用户和单个好友的关系）

   - id
   - 用户本人
   - 好友
   - 分组（也写一个表）
   - 创建时间

4. 好友请求表（加上一个状态是等待处理就不可添加）

   - id
   - 发送申请的人
   - 接受申请的人
   - 状态（接受、等待处理、拒绝）
   - 创建时间
   - 更新时间

5. 笔记权限表

   - id
   - 笔记id
   - 被授权的用户id
   - 是否可编辑（1-可读 2-可编辑）
   - 授权时间

6. 浏览历史表

   - id
   - 用户id
   - 查看的笔记id
   - 浏览次数
   - 查看的时间（加唯一约束，查看单个笔记只实现更新时间）

7. 文件夹表

   - id
   - 所属用户id
   - 文件夹名称
   - 创建时间
   - 更新时间
